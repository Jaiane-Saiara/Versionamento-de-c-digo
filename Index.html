# Code Versioning A short guide on code versioning with git and github based on Digital Innovation one (DIO) *** Basically **code versioning** is an organized way of managing changes to code by creating versions. For the operation of code versioning, a **version control system (VCS)** is used. For the operation of code versioning, a version control system is used. This system records the history of updates to a file, manages which updates were made, organizes and controls. There is the **centralized version control system (CVCS)** and the **distributed version control system (DVCS)**.
